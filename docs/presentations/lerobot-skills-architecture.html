LeRobot Simulation Skills
架构介绍

Isaac Sim 核心拆分 · 可插拔物理/渲染 · LeRobot 数据闭环

面向智能体的仿真数据采集与自动评测，把重型一体化仿真应用拆成可组合的 skills。

1. 背景：为什么拆分 Isaac Sim

  • Isaac Sim 是庞大的一体化软件包：仿真、渲染、UI、扩展系统、资产工具链都在同一应用栈中，组件多、依赖重、冷启动慢。
  • MuJoCo 内置渲染器适合快速调试，但材质、光照、相机效果和合成数据真实性较弱。
  • 我们的选择：保留 Isaac/Omniverse 里真正需要的核心能力，拆成物理仿真、RTX 渲染、数据记录、策略评测四类可组合 skill。

2. 目标：Isaac Sim 核心拆分 + LeRobot 数据格式

统一数据契约

以 LeRobot Dataset 作为训练、可视化、回放和评测的统一输入/输出格式。

低耦合通信

用 ROS 2 topic 连接 physics、render、record、rollout，各 skill 可独立启动和替换。

可插拔物理

当前使用 standalone ovphysx，后续可替换为其他物理后端。

可插拔渲染

当前使用 omni.hydra.rtx，后续保留 Blender/离线渲染接入空间。

3. 总体架构

USD Robot / Scene

机器人、本体、传感器、场景资产

→

lerobot-serve

ovphysx 物理仿真；发布 joint state、TF、metadata、clock；订阅 command

lerobot-render

订阅 TF；用 Omni RTX 渲染多相机图像；发布 image topic

→

record / rollout

记录 demonstration 或加载 policy 闭环评测，输出 LeRobot dataset

核心协议：/lerobot/joint_states、/lerobot/command、/tf、/lerobot/render/<sensor>/image_raw、LeRobot dataset。

4. lerobot-serve：可插拔物理仿真

  • 当前实现使用 standalone ovphysx：加载 USD articulation，创建 PhysX tensor binding。
  • 绑定 DOF position / velocity / target / effort，以及 root pose、link pose。
  • ROS 合约：发布 /lerobot/joint_states、/lerobot/root_pose、/tf、/lerobot/metadata、/clock。
  • 订阅 /lerobot/command，按关节名把 position / velocity / effort 写回 ovphysx target。
  • MOZ01 可通过 --control-profile moz01 展开四连杆夹爪耦合；多 articulation 场景可使用 <root>::<joint> 命名。

5. lerobot-render：可插拔渲染器

  • 当前使用 Omniverse Kit + omni.hydra.rtx，不启动完整 Isaac Sim app。
  • launcher 自动寻找 kit.exe，注入 skill app / extension，并通过 setting 传入 USD、相机、分辨率、端口。
  • extension 打开 USD 后切到 session layer，避免改写源资产；若场景无灯光，临时添加 dome/key light。
  • 自动发现 USD Camera 或读取 sensors JSON；ROS bridge 订阅 /tf，发布 /lerobot/render/<sensor>/image_raw。
  • 已尝试兼容 Blender 作为替代渲染后端，但现阶段帧率太低，暂不作为默认实时采集路径。

6. lerobot-record-simulation：自动采集 LeRobot 数据

  • 目标：自动拉起或连接 serve 和 render，将仿真 demonstration 写成本地 LeRobot dataset。
  • 记录字段：observation.state、action、observation.images.<sensor>、task。
  • 支持 --sensors auto|none|name1,name2：可自动发现 render image topics，也可录制纯 proprioception 数据。
  • 每个 episode 随机 reset 到 goal 附近，再用 smoothstep 轨迹生成专家动作。
  • 每集 save_episode()，结束 finalize()，再重新打开 dataset 做校验，避免产出不可读数据。

7. lerobot-rollout-simulation：仿真环境自动评测

  1. 自动拉起或连接 serve 和 render，发现 joints 和 sensors，创建与当前仿真一致的 LeRobot features。
  2. 加载本地或 Hugging Face LeRobot checkpoint，以及 preprocessor / postprocessor。
  3. 每个 episode 随机 reset：observation → policy inference → action → ROS command。
  4. 保存 rollout dataset，并写 evaluation.json。

指标包括 reset error、final/minimum goal error、success、inference latency、control overrun、success rate、P95 latency 等。

8. 后续扩展路线

moz-robot-setup oven 场景搭建 资产重建 skill 渲染增强 skill

  • 本体配置 skill：robot morphology、关节命名、control profile、传感器挂载。
  • 场景搭建 skill：任务场景、交互物体、随机化参数、评测目标。
  • 资产重建 skill：从扫描/图片/网格重建 USD asset，并补齐碰撞、材质、尺度、语义信息。
  • 渲染增强 skill：材质增强、光照随机化、domain randomization、合成数据质量检查。
